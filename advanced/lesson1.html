Объектно-ориентированное программирование

ООП — это paradigma программирования, где программа состоит из объектов. Это делает код структурированным, понятным и лёгким для расширения.

Что такое класс и объект

Представьте, что вы создаёте игру. В ней есть игрок, враги, предметы. У каждого есть свойства (здоровье, имя, цвет) и действия (атаковать, двигаться, собирать).

Ключевые понятия

Класс — это чертёж, шаблон. Например, чертёж "Автомобиль".
Объект — это конкретный экземпляр класса. Например, "красная Toyota Camry 2020 года".

Рассмотрим на простом примере:

main.py Пример 1: Простой класс 
# Создаём класс Dog (Собака)
class Dog:
    # Метод __init__ вызывается при создании объекта
    def __init__(self, name, age):
        self.name = name  # Атрибут объекта
        self.age = age    # Атрибут объекта
    
    # Метод — функция внутри класса
    def bark(self):
        return f"{self.name} говорит: Гав-гав!"

# Создаём объекты (экземпляры класса)
dog1 = Dog("Бобик", 3)
dog2 = Dog("Шарик", 5)

# Обращаемся к атрибутам и методам
print(f"Имя: {dog1.name}, Возраст: {dog1.age}")
print(dog1.bark())
print()
print(f"Имя: {dog2.name}, Возраст: {dog2.age}")
print(dog2.bark())

Вывод программы:

Терминал Результат 
Имя: Бобик, Возраст: 3
Бобик говорит: Гав-гав!

Имя: Шарик, Возраст: 5
Шарик говорит: Гав-гав!
Важно запомнить:
  • __init__ — специальный метод, который вызывается автоматически при создании объекта
  • self — ссылка на текущий объект, позволяет обращаться к его атрибутам
  • Атрибуты (self.name) — данные объекта
  • Методы (bark()) — действия объекта

Атрибуты класса и объекта

Атрибуты могут быть общими для всех объектов (атрибуты класса) и уникальными для каждого (атрибуты экземпляра).

main.py Пример 2: Атрибуты класса и объекта 
class Car:
    # Атрибут класса — общий для всех машин
    wheels = 4
    
    def __init__(self, brand, model, color):
        # Атрибуты экземпляра — уникальные для каждой машины
        self.brand = brand
        self.model = model
        self.color = color
    
    def get_info(self):
        return f"{self.color} {self.brand} {self.model}"

# Создаём разные машины
car1 = Car("Toyota", "Camry", "Красная")
car2 = Car("BMW", "X5", "Чёрный")

print(f"Атрибут класса (колёса у всех): {Car.wheels}")
print(f"car1: {car1.get_info()}")
print(f"car2: {car2.get_info()}")

# Можно изменить атрибут класса для всех
Car.wheels = 6
print(f"\nПосле изменения: у car1 теперь {car1.wheels} колёс")

Вывод программы:

Терминал Результат 
Атрибут класса (колёса у всех): 4
car1: Красная Toyota Camry
car2: Чёрный BMW X5

После изменения: у car1 теперь 6 колёс

Инкапсуляция

Инкапсуляция — это механизм, который скрывает внутреннее состояние объекта и требует изменения только через публичные методы.

Уровни доступа в Python

public (публичный) — атрибут доступен везде
_protected (защищённый) — одно подчёркивание, доступ внутри класса и наследников
__private (приватный) — два подчёркивания, доступ только внутри класса

main.py Пример 3: Инкапсуляция 
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner           # публичный
        self._balance = balance      # защищённый
        self.__password = "1234"     # приватный
    
    def deposit(self, amount):
        """Пополнить счёт"""
        if amount > 0:
            self._balance += amount
            return f"Пополнено на {amount}. Баланс: {self._balance}"
        return "Некорректная сумма"
    
    def withdraw(self, amount):
        """Снять деньги"""
        if 0 < amount <= self._balance:
            self._balance -= amount
            return f"Снято {amount}. Остаток: {self._balance}"
        return "Недостаточно средств"
    
    def get_balance(self):
        """Публичный метод для получения баланса"""
        return f"Баланс счёта: {self._balance}"

# Создаём счёт
account = BankAccount("Иван", 1000)

print(account.deposit(500))
print(account.withdraw(200))
print(account.get_balance())

# Попытка доступа к приватному атрибуту вызовет ошибку:
# print(account.__password)  # AttributeError!

Вывод программы:

Терминал Результат 
Пополнено на 500. Баланс: 1500
Снято 200. Остаток: 1300
Баланс счёта: 1300

Наследование

Наследование позволяет создавать новый класс на основе существующего. Дочерний класс наследует все атрибуты и методы родительского.

main.py Пример 4: Наследование 
# Родительский класс
class Animal:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def speak(self):
        return "Какой-то звук"
    
    def info(self):
        return f"{self.name}, {self.age} лет"

# Дочерний класс наследует от Animal
class Cat(Animal):
    # Переопределяем метод родителя
    def speak(self):
        return "Мяу!"
    
    def scratch(self):
        return f"{self.name} точит когти"

# Ещё один дочерний класс
class Dog(Animal):
    def speak(self):
        return "Гав!"
    
    def fetch(self):
        return f"{self.name} приносит мяч"

# Создаём объекты
cat = Cat("Барсик", 2)
dog = Dog("Бобик", 4)

print(f"Кот: {cat.info()}")
print(cat.speak())
print(cat.scratch())
print()
print(f"Собака: {dog.info()}")
print(dog.speak())
print(dog.fetch())

Вывод программы:

Терминал Результат 
Кот: Барсик, 2 лет
Мяу!
Барсик точит когти

Собака: Бобик, 4 лет
Гав!
Бобик приносит мяч
Полиморфизм — это возможность использовать объекты разных классов через общий интерфейс. В примере выше оба класса имеют метод speak(), но реализация разная.

Практический пример: Система управления сотрудниками

Создадим полноценную систему для компании с разными типами сотрудников.

main.py Пример 5: Реальный проект 
class Employee:
    """Базовый класс для всех сотрудников"""
    
    def __init__(self, name, employee_id, base_salary):
        self.name = name
        self.employee_id = employee_id
        self._base_salary = base_salary
    
    def get_salary(self):
        return self._base_salary
    
    def info(self):
        return f"ID: {self.employee_id}, Имя: {self.name}"

class Manager(Employee):
    """Менеджер с бонусом"""
    
    def __init__(self, name, employee_id, base_salary, bonus):
        super().__init__(name, employee_id, base_salary)
        self.bonus = bonus
    
    def get_salary(self):
        return self._base_salary + self.bonus

class Developer(Employee):
    """Разработчик с премией за проекты"""
    
    def __init__(self, name, employee_id, base_salary, projects_completed):
        super().__init__(name, employee_id, base_salary)
        self.projects_completed = projects_completed
    
    def get_salary(self):
        # Премия 10% за каждый проект
        return self._base_salary + (self._base_salary * 0.1 * self.projects_completed)

# Создаём сотрудников
manager = Manager("Анна Петрова", "M001", 80000, 20000)
dev1 = Developer("Иван Иванов", "D001", 100000, 3)
dev2 = Developer("Ольга Сидорова", "D002", 120000, 5)

employees = [manager, dev1, dev2]

# Выводим информацию
print("=== Зарплатная ведомость ===\n")
for emp in employees:
    print(f"{emp.info()}")
    print(f"Зарплата: {emp.get_salary():,.0f} руб.")
    print("-" * 30)

Вывод программы:

Терминал Результат 
=== Зарплатная ведомость ===

ID: M001, Имя: Анна Петрова
Зарплата: 100,000 руб.
------------------------------
ID: D001, Имя: Иван Иванов
Зарплата: 130,000 руб.
------------------------------
ID: D002, Имя: Ольга Сидорова
Зарплата: 180,000 руб.
------------------------------
super() — функция для вызова методов родительского класса. В примере super().__init__(...) вызывает конструктор класса Employee.

Магические методы

Магические методы (dunder methods) начинаются и заканчиваются двойным подчёркиванием. Они позволяют изменить поведение объектов.

main.py Пример 6: Магические методы 
class Book:
    def __init__(self, title, author, pages):
        self.title = title
        self.author = author
        self.pages = pages
    
    # Строковое представление для пользователя
    def __str__(self):
        return f"'{self.title}' by {self.author}"
    
    # Представление для разработчика
    def __repr__(self):
        return f"Book('{self.title}', '{self.author}', {self.pages})"
    
    # Количество страниц
    def __len__(self):
        return self.pages
    
    # Сравнение по количеству страниц
    def __gt__(self, other):
        return self.pages > other.pages

book1 = Book("Python Basics", "John Smith", 250)
book2 = Book("Advanced Python", "Jane Doe", 450)

print(book1)              # Вызывает __str__
print(repr(book2))        # Вызывает __repr__
print(f"Страниц в книге 1: {len(book1)}")
print(f"Книга 2 больше книги 1: {book2 > book1}")

Вывод программы:

Терминал Результат 
'Python Basics' by John Smith
Book('Advanced Python', 'Jane Doe', 450)
Страниц в книге 1: 250
Книга 2 больше книги 1: True
