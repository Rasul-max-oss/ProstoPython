Работа с API и JSON

API позволяет программам общаться друг с другом. JSON — формат данных, который используют почти все современные API. Научимся получать данные из интернета и работать с ними.

Что такое JSON

JSON (JavaScript Object Notation) — текстовый формат обмена данными. Он лёгкий для чтения и понятен как человеку, так и машине.

Структура JSON

JSON состоит из пар "ключ: значение" и массивов.
Объект — фигурные скобки {}
Массив — квадратные скобки []

data.json Пример JSON 
{
  "name": "Иван Петров",
  "age": 28,
  "city": "Москва",
  "isDeveloper": true,
  "skills": ["Python", "JavaScript", "SQL"],
  "projects": [
    {
      "title": "Веб-сайт",
      "year": 2023
    },
    {
      "title": "Telegram-бот",
      "year": 2024
    }
  ]
}

Работа с JSON в Python

В Python есть встроенный модуль json для работы с этим форматом.

main.py Пример 1: Чтение JSON 
import json

# JSON-строка (как будто получили от API)
json_string = '''
{
  "name": "Анна Смирнова",
  "age": 25,
  "city": "Санкт-Петербург",
  "skills": ["Python", "Django", "PostgreSQL"]
}
'''

# Преобразуем JSON в словарь Python
data = json.loads(json_string)

# Теперь работаем как с обычным словарём
print(f"Имя: {data['name']}")
print(f"Возраст: {data['age']}")
print(f"Город: {data['city']}")
print("Навыки:", ", ".join(data['skills']))

Вывод программы:

Терминал Результат 
Имя: Анна Смирнова
Возраст: 25
Город: Санкт-Петербург
Навыки: Python, Django, PostgreSQL
json.loads() — преобразует JSON-строку в объект Python (словарь).
json.load() — читает JSON из файла.

Преобразование Python в JSON

Часто нужно отправить данные в формате JSON. Для этого используется json.dumps().

main.py Пример 2: Создание JSON 
import json

# Словарь Python
user = {
    "id": 1,
    "name": "Дмитрий Козлов",
    "email": "dmitry@example.com",
    "active": True,
    "roles": ["admin", "user"]
}

# Преобразуем в JSON-строку
json_string = json.dumps(user, ensure_ascii=False, indent=2)

print("JSON-строка:")
print(json_string)

# Сохраняем в файл
with open("user.json", "w", encoding="utf-8") as f:
    json.dump(user, f, ensure_ascii=False, indent=2)

print("\nДанные сохранены в user.json")

Вывод программы:

Терминал Результат 
JSON-строка:
{
  "id": 1,
  "name": "Дмитрий Козлов",
  "email": "dmitry@example.com",
  "active": true,
  "roles": ["admin", "user"]
}

Данные сохранены в user.json
ensure_ascii=False — сохраняет кириллицу в исходном виде, а не преобразует в \u0435
indent=2 — делает вывод красивым с отступами

Что такое API

API (Application Programming Interface) — набор правил, позволяющий программам получать данные от других сервисов.

REST API

Самый популярный тип API. Использует HTTP-запросы:
GET — получить данные
POST — создать данные
PUT/PATCH — обновить данные
DELETE — удалить данные

Рассмотрим работу с API на примере бесплатного сервиса JSONPlaceholder (тестовое API).

main.py Пример 3: GET-запрос к API 
import requests

# Получаем данные о пользователе
response = requests.get("https://jsonplaceholder.typicode.com/users/1")

# Проверяем статус ответа
if response.status_code == 200:
    # Преобразуем ответ в JSON
    user = response.json()
    
    print("Данные пользователя:")
    print(f"Имя: {user['name']}")
    print(f"Email: {user['email']}")
    print(f"Город: {user['address']['city']}")
    print(f"Компания: {user['company']['name']}")
else:
    print(f"Ошибка: {response.status_code}")

Вывод программы:

Терминал Результат 
Данные пользователя:
Имя: Leanne Graham
Email: Sincere@april.biz
Город: Gulfton
Компания: Romaguera-Crona
Для работы нужен модуль requests. Установите:
pip install requests

Отправка данных через POST

POST-запрос создаёт новые данные на сервере.

main.py Пример 5: POST-запрос 
import requests

# Данные для отправки
new_post = {
    "title": "Мой первый пост",
    "body": "Это текст поста, созданного через API",
    "userId": 1
}

# Отправляем POST-запрос
response = requests.post(
    "https://jsonplaceholder.typicode.com/posts",
    json=new_post
)

# Получаем ответ
if response.status_code == 201:
    created = response.json()
    print("Пост создан!")
    print(f"ID: {created['id']}")
    print(f"Заголовок: {created['title']}")
    print(f"Текст: {created['body']}")
else:
    print(f"Ошибка: {response.status_code}")

Вывод программы:

Терминал Результат 
Пост создан!
ID: 101
Заголовок: Мой первый пост
Текст: Это текст поста, созданного через API
JSONPlaceholder — тестовое API, данные не сохраняются реально. Но ответ имитирует успешное создание.

Работа с параметрами запроса

API позволяют фильтровать данные через параметры.

import requests

# Параметры для фильтрации
params = {
    "userId": 1,      # только посты пользователя 1
    "_limit": 3       # ограничить 3 результатами
}

# GET-запрос с параметрами
response = requests.get(
    "https://jsonplaceholder.typicode.com/posts",
    params=params
)

posts = response.json()

print(f"Посты пользователя #1 (первые 3):\n")
for post in posts:
    print(f"ID: {post['id']}")
    print(f"Заголовок: {post['title']}")
    print()

Обработка ошибок

При работе с API важно обрабатывать возможные ошибки.

main.py Пример 7: Обработка ошибок 
import requests

def get_user(user_id):
    """Безопасное получение данных пользователя"""
    url = f"https://jsonplaceholder.typicode.com/users/{user_id}"
    
    try:
        response = requests.get(url, timeout=5)
        
        # Проверка статуса
        if response.status_code == 404:
            return f"Пользователь {user_id} не найден"
        elif response.status_code != 200:
            return f"Ошибка сервера: {response.status_code}"
        
        # Парсинг JSON
        user = response.json()
        return f"Пользователь: {user['name']}, Email: {user['email']}"
    
    except requests.exceptions.Timeout:
        return "Превышено время ожидания ответа"
    except requests.exceptions.ConnectionError:
        return "Ошибка подключения к сети"
    except json.JSONDecodeError:
        return "Ошибка чтения JSON"

# Тестируем
print(get_user(1))      # Существующий
print(get_user(999))    # Не существует
print(get_user("abc"))  # Некорректный ID

Вывод программы:

Терминал Результат 
Пользователь: Leanne Graham, Email: Sincere@april.biz
Пользователь 999 не найден
Пользователь abc не найден

Практический проект: Погодное приложение

Создадим программу, которая получает погоду из API OpenWeatherMap.

Для работы нужен API-ключ. Зарегистрируйтесь на openweathermap.org и получите бесплатный ключ.
weather.py Пример 8: Погодное приложение 
import requests

API_KEY = "ваш_api_ключ"  # Замените на свой ключ
BASE_URL = "https://api.openweathermap.org/data/2.5/weather"

def get_weather(city):
    """Получить погоду для города"""
    params = {
        "q": city,
        "appid": API_KEY,
        "units": "metric",      # Цельсии
        "lang": "ru"            # Русский язык
    }
    
    try:
        response = requests.get(BASE_URL, params=params)
        response.raise_for_status()  # Вызовет ошибку при статусе != 200
        
        data = response.json()
        
        # Извлекаем данные
        temp = data["main"]["temp"]
        feels_like = data["main"]["feels_like"]
        humidity = data["main"]["humidity"]
        description = data["weather"][0]["description"]
        wind_speed = data["wind"]["speed"]
        
        print(f"\nПогода в городе {city}")
        print("=" * 30)
        print(f"Температура: {temp}°C (ощущается как {feels_like}°C)")
        print(f"Описание: {description.capitalize()}")
        print(f"Влажность: {humidity}%")
        print(f"Ветер: {wind_speed} м/с")
        
    except requests.exceptions.HTTPError as e:
        if e.response.status_code == 404:
            print(f"Город '{city}' не найден")
        else:
            print(f"Ошибка API: {e}")
    except Exception as e:
        print(f"Произошла ошибка: {e}")

# Запуск
get_weather("Moscow")
get_weather("London")
get_weather("New York")
