Основы ООП

Объектно-ориентированное программирование — парадигма, где программа состоит из объектов. Изучим классы, объекты и основные принципы ООП.

1. Классы и объекты

Класс — это чертёж объекта. Объект — это экземпляр класса.

class_basic.py
# Создание класса
class Dog:
    def __init__(self, name, age):
        self.name = name  # Атрибут объекта
        self.age = age
    
    def bark(self):
        print(f"{self.name} говорит: Гав!")

# Создание объектов
dog1 = Dog("Бобик", 3)
dog2 = Dog("Шарик", 5)

dog1.bark()  # Бобик говорит: Гав!
dog2.bark()  # Шарик говорит: Гав!
self: Ссылка на текущий объект. Должна быть первым параметром методов класса.

2. Атрибуты и методы

Атрибуты — это данные объекта. Методы — это функции объекта.

attributes_methods.py
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def greet(self):
        return f"Привет, я {self.name}"
    
    def birthday(self):
        self.age += 1
        return f"Мне теперь {self.age} лет"

person = Person("Алекс", 25)
print(person.greet())    # Привет, я Алекс
print(person.birthday()) # Мне теперь 26 лет

3. Наследование

Наследование позволяет создать новый класс на основе существующего.

inheritance.py
class Animal:
    def __init__(self, name):
        self.name = name
    
    def speak(self):
        print("Звук...")

class Cat(Animal):  # Cat наследует Animal
    def speak(self):
        print(f"{self.name} говорит: Мяу!")

class Dog(Animal):
    def speak(self):
        print(f"{self.name} говорит: Гав!")

cat = Cat("Мурка")
dog = Dog("Бобик")

cat.speak()  # Мурка говорит: Мяу!
dog.speak()  # Бобик говорит: Гав!
Переопределение: Дочерний класс может переопределить метод родительского класса.

4. Инкапсуляция

Инкапсуляция скрывает внутренние данные объекта. В Python используется соглашение об подчёркивании.

encapsulation.py
class BankAccount:
    def __init__(self, balance):
        self._balance = balance  # Защищённый атрибут
    
    def deposit(self, amount):
        if amount > 0:
            self._balance += amount
    
    def withdraw(self, amount):
        if amount <= self._balance:
            self._balance -= amount
    
    def get_balance(self):
        return self._balance

account = BankAccount(1000)
account.deposit(500)
account.withdraw(200)
print(account.get_balance())  # 1300

5. Практические примеры

Пример 1

Класс Книга

Моделирование книги

class Book:
    def __init__(self, title, author, pages):
        self.title = title
        self.author = author
        self.pages = pages
        self.current_page = 1
    
    def read(self, pages):
        self.current_page += pages
        if self.current_page > self.pages:
            self.current_page = self.pages
    
    def get_progress(self):
        return f"Прочитано {self.current_page}/{self.pages} страниц"

book = Book("Python для начинающих", "Иван Петров", 300)
book.read(50)
print(book.get_progress())
Пример 2

Класс Rectangle

Геометрическая фигура

class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    def area(self):
        return self.width * self.height
    
    def perimeter(self):
        return 2 * (self.width + self.height)
    
    def is_square(self):
        return self.width == self.height

rect = Rectangle(5, 10)
print(f"Площадь: {rect.area()}")
print(f"Периметр: {rect.perimeter()}")
print(f"Квадрат: {rect.is_square()}")
Пример 3

Класс Student

Ученик с оценками

class Student:
    def __init__(self, name):
        self.name = name
        self.grades = []
    
    def add_grade(self, grade):
        self.grades.append(grade)
    
    def average(self):
        if self.grades:
            return sum(self.grades) / len(self.grades)
        return 0

student = Student("Мария")
student.add_grade(5)
student.add_grade(4)
student.add_grade(5)
print(f"Средний балл: {student.average():.2f}")
Полиморфизм: Разные классы могут иметь методы с одинаковым именем. Это позволяет обрабатывать объекты разных классов единообразно.
